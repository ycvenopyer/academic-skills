# 论文分析维度参考

## 分析框架（10个维度）

### 1. 研究问题与动机
- 论文试图解决什么问题？
- 为什么这个问题重要？
- 现有方法有什么不足？

### 2. 方法与创新点
- 核心方法是什么？
- 创新点在哪里（算法/架构/应用场景）？
- 与技术现状相比有什么突破？

### 3. 实验设计
- 使用了哪些数据集？规模如何？
- 基线方法是否合理？
- 评估指标是否全面？

### 4. 实验结果
- 主要结果如何？是否显著优于基线？
- 是否包含消融实验？结论是否明确？
- 统计显著性是否报告？

### 5. 理论分析
- 是否有理论证明或收敛性分析？
- 复杂度分析（时间/空间）？
- 方法的局限性是否讨论？

### 6. 可复现性
- 代码是否开源？
- 超参数设置是否详述？
- 实验环境是否明确？

### 7. 写作质量
- 论文结构是否清晰？
- 图表是否有效传达信息？
- 公式推导是否完整？

### 8. 相关工作定位
- 对相关工作的覆盖是否全面？
- 与现有方法的对比是否客观？
- 引用是否合理？

### 9. 局限性讨论
- 论文是否讨论了方法的局限性？
- 是否有失败的实验案例？
- 未来工作方向是否明确？

### 10. 综合评估
- 论文的优缺点总结
- 对该领域的贡献程度
- 对本人研究的启发

## 输出格式

分析完成后，以结构化Markdown输出：

```markdown

## 1. 基本信息
[标题、作者、会议/期刊、年份]

## 2. 研究问题与动机
...

## 3. 核心方法
...

## 4. 实验与结果
...

## 5. 优缺点分析
### 优点
- ...

### 不足
- ...

## 6. 个人思考
- 对本人研究的启发
- 可改进的方向
```
